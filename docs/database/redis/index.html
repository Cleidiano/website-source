Redis - L2 cache

To use Redis for L2 caching set the ebean.test.redis property to latest in application-test.yaml like the example below. This will automatically start a redis docker container for use with L2 caching. 

ebean:
  test:
    redis: latest
    platform: h2 # h2, postgres, mysql, oracle, sqlserver, sqlite
    ddlMode: dropCreate # none | dropCreate | migrations | create
    dbName: my_app

ebean-redis dependency

Add ebean-redis as a dependency. ebean-redis will automatically register itself with ebean as the L2 cache service. 

  
    io.ebean
    ebean-redis
    12.1.1

configuration

To configure the redis server to use in main/resources/application.yaml we configure the following properties: 

ebean:
  redis:
    server: test-server
    port: 99
    minIdle: 2
    maxIdle: 20
    maxTotal: 20
    maxWaitMillis: 300

@Cache

Entity beans annotated with @Cache will now use redis for caching. 


@Cache(enableQueryCache = true, nearCache = true, naturalKey = "name")
@Entity
public class Person extends EBase {

  public enum Status {
    NEW,
    ACTIVE,
    INACTIVE
  }

  @Index(unique = true)
  String name;

  Status status;

  LocalDate localDate;

  String notes;

  public Person(String name) {
    this.name = name;
    this.status = Status.NEW;
  }
  ...
}
